## 발견 사항 — 범위 대부분이 이미 완료돼 있었음

Phase 6 착수 시점에 `web/` 전체를 조사한 결과, 궁합방(멀티플레이)·리퍼럴 포인트·연애/직업 메뉴 배선은 **Phase 3 웹 스캐폴드 작업(phase3-web) 때 이미 함께 이식·영어화돼 있었음**을 확인:

- `web/lib/rooms.ts`(방 생성·제출·상한 100·닉네임 upsert), `web/lib/retention.ts`(방 유예 스윕), `web/app/menu/compat/room/**`(생성·대시보드·조인·상세), `web/components/Room*.tsx`, `web/app/library/**`(방 목록·연장·삭제) — 전부 영어, 테스트 그린.
- `web/lib/referral.ts`·`referral-code.ts`·`points.ts`·`ref-cookie.ts`, `web/app/invite/**` — 전부 영어, 테스트 그린.
- `web/components/MenuGrid.tsx`·`web/app/menu/[slug]/page.tsx`에 `love-marriage`/`career` 완전 배선(라벨·설명·페이지 분기).
- `supabase/migrations/0001_init.sql`에 `compat_rooms`/`compat_room_entries`/`points_ledger`/`share_cards` 전 테이블이 이미 통합돼 있음.
- 한글 누출 스캔(`web/app`·`web/lib`·`web/components` 전수) — 결과 2건, 둘 다 의도된 것: `CompatScoreCard.tsx`의 `TIER_LABEL` 레코드 키(엔진 `CompatTier` 타입 자체가 한글 라벨), `engine.ts`의 문서화된 성별 인터롭 경계 주석.

**실제로 비어 있던 것**: 공유 카드 / 동적 OG 이미지. `web/app/s/**`가 전부 자리만 잡혀 있고 실제로는 정적 플레이스홀더("A friend read their Korean fortune on Dotori")를 렌더링 중이었음 — 이유는 `src/reading/character.ts`(공유 카드용 일간 캐릭터 10종)가 Phase 0에서 verbatim 복사된 100% 한국어 콘텐츠라, 포팅 에이전트가 한글 누출을 막기 위해 의도적으로 기능을 스텁 처리해뒀던 것(`"Phase-6 scope reduction"` 주석으로 명시돼 있었음). 궁합 카드 쪽은 `tierTeaserEn`이 이미 Phase 2 마무리 때 만들어져 있어 스텁 사유가 이미 stale했음.

## 한 일 — 공유 카드 실기능 구현

`executor`(opus) 서브에이전트에 위임, 완료 후 직접 diff 검토 + `code-reviewer`(opus) 독립 리뷰 통과.

1. **`src/reading/character-en.ts`(신규, 엔진 레이어)**: `character.ts`(verbatim, 수정 금지)의 영어 형제 파일. 일간 10종 + 폴백을 승인된 "따뜻한 절친" 톤으로 재창작(직역 아님). `characterForDayStemEn`/`stemCardLabelEn`(`STEM_GLOSSARY` 재사용)/`elementCountToEn` export.
2. **`web/lib/share-cards.ts`(신규)**: `SoloCardPayload`/`CompatCardPayload` 타입, `buildSoloPayload`/`buildCompatPayload`(기존 `tierTeaserEn`·`CompatScoreCard.TIER_LABEL`·`character-en` 재사용, 신규 문구 없음), `truncateForCard(text, maxChars)` — 단어 경계 우선 절단 + 말줄임표, 영어 문장이 한국어 원본보다 길어서 생긴 요구사항.
3. **`web/app/s/actions.ts` 재배선**: 클라이언트가 보낸 표시값을 신뢰하지 않고 식별자(출생 쿼리 또는 roomId+entryId)에서 서버가 직접 재계산(`buildChart`/`compatScoreForEntry`) — 위조 방지 원칙 준수.
4. **`ShareCardView.tsx`/`s/[id]/page.tsx`**: 실제 solo/compat 페이로드 렌더(캐릭터명·일간 라벨·오행 바 / 점수·티어·host♥guest), 페이로드 손상 시 기존 제네릭 브랜드 카드로 무중단 폴백. `generateMetadata`도 카드별 실데이터로 교체.
5. **`s/[id]/opengraph-image.tsx`**: 정적 플레이스홀더 → `next/og` 실제 동적 렌더(솔로 스플릿 패널 / 궁합 점수 중심), 폰트 번들 없이 satori 기본 폰트로 라틴 텍스트 커버. `truncateForCard`를 캐릭터·티저 라인에 적용.
6. **리뷰에서 발견해 직접 수정**: OG 이미지의 host/guest 닉네임(사용자 입력, 상한 길이 없음)에 `truncateForCard` 미적용이었던 것을 발견 → `NAME_MAX=24`로 캡. 점수/티어 카드의 유일한 실제 오버플로 위험 필드였음.
7. 신규 테스트 5개 파일: `character-en.test.ts`(10종+폴백, 한글 누출 없음), `share-cards.test.ts`(`truncateForCard` 경계값), `web/app/__tests__/actions.test.ts`(실제 재계산 페이로드 검증), 기존 `share-card-view.test.tsx` 갱신.
