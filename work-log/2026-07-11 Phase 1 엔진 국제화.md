# Phase 1 — 엔진 국제화 (international-birth, glossary, guard EN)

## 배경

Phase 0에서 부트스트랩한 Dotori(K-saju) 엔진 코어는 한국 출생만 처리 가능했고(`manseryeok-adapter.ts`의
`KOREA_DST_PERIODS` 기반 보정), 프롬프트 스텁은 영어였지만 정작 차트 요약(`chart-summary.ts`)은 여전히
한글을 생성해 LLM에 전달하고 있었다. 이번 Phase 1은 (1) 전 세계 어디서 태어났든 정확한 사주 계산이
가능하도록 엔진을 확장하고, (2) 엔진 산출물(한자·한글 라벨)을 영어로 옮기는 완전한 경로를 만들고,
(3) 영어 안전 가드를 갖추는 것을 목표로 했다.

## 변경 사항

### 1. `src/chart-input/international-birth.ts` (신규)
- IANA 타임존(`timeZone`) + 현지 시각 + 실제 출생지 경도를 입력받아 `SajuChart`를 산출.
- 핵심 원리: `calculateSaju()`는 입력 시각을 "KST(UTC+9) civil clock"로 간주하고, 경도로
  `(longitude-135)/15h`의 진태양시 보정을 추가로 적용한다. 따라서
  `true_solar_time = UTC_instant + longitude/15h` 라는, 어느 출생지에나 통하는 시간대-무관 공식이
  성립 — "정확한 UTC 순간"과 "실제 경도"만 구하면 전 세계 어디든 동일한 방식으로 계산 가능함을 도출.
- `Intl.DateTimeFormat`로 IANA 타임존의 실제(DST 포함) 오프셋을 순간 단위로 계산 → 로컬 시각을 UTC로
  역산 → UTC+9h로 KST-equivalent 변환 → `calculateSaju(..., { longitude, applyTimeCorrection: true })`.
- `manseryeok-adapter.ts`의 `KOREA_DST_PERIODS`/`adjustForDst`는 한국 자체의 1948~88년 역사적
  일광절약시간만을 위한 것이라 완전히 우회(bypass) — 이미 IANA 타임존으로 정확히 변환된 시각에
  다시 적용하면 이중 보정이 됨.
- `manseryeok-adapter.ts`는 절대 수정하지 않는 파일이라(ENGINE_SYNC.md 규칙), 그 파일의 비공개
  `pillarFromHanja()` 헬퍼를 그대로 복제해 사용(제네릭 로직, 한국 특수 규칙 아님).
- 골든 테스트(12개, `__tests__/international-birth.test.ts`): New York(겨울 EST) / London(여름
  BST) / Berlin(겨울 CET) / Los Angeles(날짜 경계를 넘는 케이스) / **Seoul 1988(DST 기간)** —
  Node ICU의 `Asia/Seoul` 1988년 DST 데이터가 `KOREA_DST_PERIODS` 표와 정확히 일치함을 확인하고,
  국제 경로로 계산한 결과가 기존 `manseryeok-adapter.birthToSajuChart`(한국 전용 경로) 결과와
  **완전히 동일**함을 교차검증. 추가로 경도 -170~170 프로브(오류 없이 well-formed 차트 산출 확인).

### 2. `src/i18n/glossary-en.ts` (스켈레톤 → 완전판)
- 10천간(+음양+오행), 12지지(동물), 5오행, 10십신, 12운성, 5신살, 음양 — 엔진이 생성 가능한 모든
  한글/한자 라벨을 영어로 매핑.
- **중요 발견**: '양'이 "양(Yang) 음양"과 "양(Nurture) 12운성 이름" 두 가지로 겹쳐 쓰인다 — flat
  merge 방식이면 마지막에 spread된 카테고리가 조용히 다른 쪽을 덮어써 버그가 됨. 그래서
  `glossOf(category, koreanLabel)` 형태의 카테고리-스코프 조회 함수로 설계 변경, `GLOSSARY_EN`
  flat map은 충돌 없는 카테고리(원소/천간/지지/십신/신살)만 포함하고 음양·12운성은 제외.
- 완전성 테스트(`__tests__/glossary-en.test.ts`, 10개): `TenGod`/`TwelveState`/`SinsalCode`
  유니온 타입에 대한 컴파일타임 exhaustiveness guard(`Record<Type, true>`) + `tenGodOf()`/
  `twelveStateOf()`를 모든 천간×지지 조합에 실제로 돌려 나온 값 집합과 glossary 키 집합을 대조 —
  엔진에 새 값이 추가되면 이 테스트가 실패한다.

### 3. `src/reading/chart-summary-en.ts` (신규)
- `chart-summary.ts`(한글)와 필드 단위로 동일한 구조, glossary-en으로 완전 영어화.
- **버그 발견 및 수정**: `generate.ts`, `summary.ts`, `oracle/index.ts`, `menus/{couple,love-marriage,career}.ts`
  6개 파일이 여전히 **한글** `buildChartSummary`(`./chart-summary`)를 import하고 있었다 — Phase 0에서

  `buildChartSummaryEn`(`./chart-summary-en`)로 교체해 엔진 국제화의 실제 배선을 완성.
