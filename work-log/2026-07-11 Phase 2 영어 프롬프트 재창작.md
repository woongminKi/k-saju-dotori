# Phase 2 — 영어 프롬프트 5종 재창작

## 배경

Phase 0/1에서 만든 영어 스텁 프롬프트(frame-en-v1, modules-en, teaser-en, menus/prompts-en,
oracle prompt-en/questions-en)는 형태만 갖춘 placeholder였다. Phase 2는 오너가 승인한 보이스
샘플을 기준으로 실제 프로덕션 프롬프트를 새로 저작하는 작업 — 번역이 아니라 타겟 독자(자기
별자리는 알지만 사주는 처음 들어보는 미국/유럽 10대 후반~30대 여성)를 위한 재창작.

## 승인된 보이스 반영

- 태그라인·리딩 발췌·오라클 답변 예시를 `frame-en-v1.ts`의 `TONE_BLOCK`에 **그대로 예시 문구로
  포함**시켜, 모델이 추상적 설명이 아니라 정확한 말투를 직접 보고 따라 쓰게 함.
- 2인칭·구어체(축약형 OK)·과한 슬랭 금지, entertainment-first 프레이밍(단정적 예측 금지),
  high/mid/low 신뢰도 헤징, "약점 + 즉시 실천 가능한 처방" 시그니처 무브, 용어는
  `src/i18n/glossary-en.ts` 에서만 — 이 5가지를 SHARED_SYSTEM_BLOCK1 안에 명문화.

   ⚠️주의 + 🌱실천)을 유지하되 길이 타깃만 영어에 맞게 재설정(~150-200단어).

4. `src/menus/prompts-en.ts` — love-marriage/couple/career 3개 스펙 완성(solo는 이 파일을 안 씀).
5. `src/oracle/prompt-en.ts`(버전 `2026-07-11.1-oracle-en-v1`) + `src/oracle/questions-en.ts` —
   {answer, reason} JSON 계약은 한국판과 동일하게 유지, answer에 "acorn" 프레이밍을 직접 녹임.
   **질문 뱅크는 번역이 아니라 재테마링**: 한국판 6카테고리(연애·관계/직장·커리어/금전·재물/건강/
   학업·시험/일상선택) → 새 6카테고리(Crushes & Texting 💌 / Friendship Drama 👯 신규 / Career
   Moves 💼 / Moving & New Places 🧳 / Money Moves 🪙 / Should-I Decisions 🌀) × 20문항 = 120문항,
   한국판과 동일 규모 유지. 건강·학업 카테고리는 팀이 명시한 5개 테마에 없어서 별도 유지하지 않고
   Should-I Decisions·Career Moves에 흡수.
6. `src/reading/guard.ts` — 보류했던 5번째 카테고리(school-of-thought hedging) 추가. Sinsal·
   12운성·합충처럼 프롬프트 자체가 mid/low 신뢰도로 명시한 해석에 대해 "this always means",
   "this proves", "there is no doubt" 같은 무조건적 단정만 좁게 차단(고신뢰 패턴에 대한 자신감
   있는 서술은 차단하지 않음).

## 라이브 이밸 하네스 (`tools/eval-prompts.ts`)

실제 엔진으로 산출한 4개의 다양한 실차트(서울 1990년 국내 출생 1건 + 뉴욕 1995·런던 2000·
LA 1988 국제 출생 3건, `internationalBirthToSajuChart` 실사용) × {solo 6모듈 전체, teaser,
oracle 3문항, love-marriage, career} + couple 2쌍을 실제 Anthropic API로 호출. 매 응답마다
hasCjkLeak/hasRawLeak/checkContentSafety/단어수 밴드를 검사하고 전체 전사를 `eval-output/`
(gitignore)에 기록.

**결과: 첫 실행에 50/50 전부 통과** — CJK 누출 0, 원시데이터 누출 0, 가드 위반 0, 단어수 밴드
이탈 0. 프롬프트 수정 없이 통과.

Anthropic API 키는 오너 승인(2026-07-11)에 따라 한국판 `saju/.env`의 값을 그대로
`K-saju/.env`로 복사(커밋 전후 `git check-ignore`로 gitignore 확인 완료).
